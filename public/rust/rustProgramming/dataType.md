# 型について

## 概要

### 基本の型(プリミティブ型)

|分類|型名|備考|
|--|--|--|
|整数型|i8, i16, i32, i64, i128, isize|符号あり（isizeはCPUのビット幅に依存）|
||u8, u16, u32, u64, u128, usize|符号なし|
|浮動小数点型|f32, f64|小数点を含む数値|
|論理値型|bool|true または false|
|文字型|char|Unicode 1文字（4バイト）|

### 複数の値をまとめる型(複合型)

- タプル(tuple)：異なる型の値を一つにまとめる
- 配列(array)：同じ型の値を固定の長さで並べる

### 独自の型

- 構造体(struct)：データの集まりに名前を付けて管理する
- 列挙型(enum)：複数の選択肢の中から一つを表す。Rustはデータを持たせることが可能
- 参照とポインタ：メモリ上の場所を指し示す型

## プリミティブ型

### 整数型（Integer）

🔢整数型には「符号あり（i）」と「符号なし（u）」があります。
  - 符号あり (i8, i16, i32, i64, i128): マイナスの値も扱えます。
  - 符号なし (u8, u16, u32, u64, u128): 0以上のプラスの値だけを扱います。
  [!NOTE]
  型名の後ろの数字は「ビット数」を表しています。例えば u8 は 8ビットなので、$0$ から $2^8 - 1$（つまり255）までの数値を格納できます。

### 浮動小数点型（Floating-Point）

🥧小数点を扱う型です。
  - f32: 単精度。メモリを節約したいときに使います。
  - f64: 倍精度。Rustのデフォルトで、精度が高いため通常はこちらを使います。

**※注意**
Rustでは型違いの変数で演算はできない。
例えば```10+3.5```のように「整数型」と「浮動小数点型」をそのまま足し算できない。厳密に型を合わせる必要がある。**（型変換について調べないとかな）**

#### 数値型の「推論」と「明示」

Rustには、プログラムを書く手間を減らすための型推論という機能がある。
- 推論に任せる: ```let x = 100;```と書くと、Rustはデフォルトで ```i32```（32ビット整数）として扱います。
- 自分で決める: ```let x: u8 = 100;``` や ```let x = 100u8;``` のように書くと、強制的に型を指定できます。

### 論理値型(bool)

```true``` か ```false``` のどちらか。
これは言わずもがな

### 文字型(char)

C言語のchar型とは異なり、1文字で4バイト(固定)である。これはUnicodeを基本にしているためである。(C言語のchar型は1バイト(Ascii))
そのため、'a'や'Z'だけでなく'あ'や'🦀'(絵文字)も1文字として扱えます。

文字の長さやバイトを知りたいとき

``` rs
let s = "あ";
println!("文字数: {}", s.chars().count()); // 1
println!("バイト数: {}", s.len());           // 3 (UTF-8での日本語は3バイト)
```

- chars().count(): 人間が感じる 「文字の数」 を返します。
- len(): メモリ上の 「バイト数」 を返します。

#### 補足：文字と文字列

少しややこしいのですが、Rustでは「1文字」と「文字列」で使い分けをしています。

- char 型（4バイト固定）:
世界中のあらゆる文字（漢字、アラビア文字、絵文字など）には、Unicodeで「コードポイント」という番号が割り振られています。この番号は最大で21ビットほど必要なため、Rustは余裕を持って4バイトで1文字を保持します。これにより、どの文字も「同じサイズ」として扱えるメリットがあります。

- String / str（UTF-8エンコード）:
一方で、たくさんの文字を並べた「文字列」の場合は、メモリを節約するために UTF-8 という形式を使います。UTF-8は、英数字なら1バイト、日本語なら3バイトというように、文字によってサイズが変わる「可変長」の仕組みです。

|種類|型|サイズ|特徴|
|:--|--|:--|
|文字||char|4バイト固定|どんな文字も1つとして等しく扱う|
|文字列|String / &str|可変長 (UTF-8)|メモリ効率が良く、ネット通信などに最適|

## 複合型

### タプル

「異なる型を一つにまとめる」型

- タプルの基本ルール 📋
  - 丸括弧 () で値を囲みます。
  - 一度サイズ（要素の数）を決めたら、後から増やしたり減らしたりすることはできません。
  - 中身の型がバラバラでもOKです。

  ``` rs
  // タプルの例：(名前, 年齢, 身長)
  let person: (&str, i32, f64) = ("Alice", 24, 160.5);
  ```

- 値の取り出し方
  - タプルの中身にアクセスするには、ドット . と インデックス（番号） を使う。番号は 0 から始まることに注意。

  ``` rs
  let name = person.0; // "Alice"
  let age = person.1;  // 24
  ```

- 「分配」して受け取る（パターンマッチング）
  - タプルをバラバラにして個別の変数に代入する「構造分解」という書き方がよく使われる。こちらの方が直感的で読みやすいコードになる。

  ``` rs
  let (name, age, height) = person;
  println!("{}さんは{}歳です。", name, age);
  ```

### 配列

「同じ型」のデータを固定数並べる仕組み。タプルとは異なり中身を混ぜることはできないが、大量のデータを整然と並べるのに適する。

- 配列の基本ルール
  - 角括弧 [] で値を囲みます。
  - サイズは固定です。一度「5個入り」と決めたら、後から要素を増やしたり減らしたりすることはできません。
  - メモリ上の連続した場所にデータが並ぶため、非常に高速に処理できます。

  ``` rs
  // [型; 要素数] という形式で書く
  let numbers: [i32; 3] = [10, 20, 30];

  // すべて同じ値で初期化する場合の便利な書き方
  let all_zeros = [0; 5]; // [0, 0, 0, 0, 0] と同じ
  ```

- 値の取り出し方
  - タプル（.0）とは違い、配列は [インデックス] を使ってアクセスします。

  ``` rs
  let first = numbers[0]; // 10
  let second = numbers[1]; // 20
  ```

- 配列外の添え字にアクセスしようとしたとき
  - メモリリークなどの原因となる配列外アクセスをRustでした場合、Rustではプログラムを安全に停止(パニック)させてくれる。

### 可変長配列(Vec<T>)

Rustで最もよく使われる「可変長配列」です。

- データが増えると、自動的にメモリ領域を拡張してくれます。
- vec.push(item) のようにして、後から要素を追加できます。
- 中身は同じ型である必要があります。

### スライス(&[T])

配列やベクタの「一部」を指し示す参照です。

- 「3番目から5番目まで」といった範囲を効率よく扱うために使います。
- サイズの情報は持っていますが、自分自身でデータを所有しているわけではなく、どこかのデータを「覗き見ている」状態です。

- スライスの特徴 
  - 所有権を持たない: データの持ち主（ベクタなど）から一時的に「参照」している状態です。
  - 型は &[T]: &（アンパサンド）がついているのは、参照であることを示しています。
  - サイズが実行時に決まる: 「何文字目から何文字目まで」という情報を持つため、柔軟に範囲を指定できます。

  ``` rs
  let numbers = [10, 20, 30, 40, 50];

  // 1番目から3番目の手前まで（20, 30）を指すスライス
  let slice = &numbers[1..3]; 

  println!("{:?}", slice); // [20, 30]
  ```
- スライスの利点
  - データの出所がどこであっても、同じように扱えること
  - 例えば、「数値のリストを受け取って合計を計算する関数」を作るとき、引数をスライス型（&[i32]）にしておけば、固定長の「配列」でも可変長の「ベクタ」でも、どちらも受け取ることができます。

### 構造体

関連する複数のデータを一つにまとめ、それぞれに名前（フィールド）を付けることができる型。

#### 名前付きフィールドを持つ構造体

- 構造体の定義と作成方法

``` rs
// 構造体の定義（設計図）
struct User {
    username: String,
    email: String,
    sign_in_count: u64,
    active: bool,
}

// 構造体のインスタンス化（実体を作る）
let user1 = User {
    email: String::from("someone@example.com"),
    username: String::from("someuser123"),
    active: true,
    sign_in_count: 1,
};
```

- フィールドへのアクセス
  - ドット . を使うことで、特定のデータだけを取り出したり、（変数が mut であれば）書き換えたりできます。

  ``` rs
  println!("ユーザー名: {}", user1.username);
  ```

- 振る舞い(関数)のセット(メソッド記法)
  - これを impl ブロック（実装ブロック）と呼びます。
  - 例えば、長方形のデータを表す構造体があったとして、その面積を計算する機能を構造体自身に持たせることができます。

  ``` rs
  // フィールド（状態、属性）
  struct Rectangle {
      width: u32,
      height: u32,
  }

  impl Rectangle {
      // 面積を計算する「メソッド」
      fn area(&self) -> u32 {
          self.width * self.height
      }
  }
  ```

  - このようなデータと機能をひとまとめにすることを「カプセル化」と呼ぶ。

### タプルと構造体の使い分けの目安

- タプル: 「関数の戻り値で2つ値を返したいだけ」など、一時的で簡単なまとまりに使う。
- 構造体: 「座標」や「ユーザー情報」など、データに明確な意味があり、何度も使い回すものに使う。
